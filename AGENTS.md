# Knovara Coding Rules

Keep the code simple and production-minded.

Before writing new code:
1. Search existing code first.
2. Reuse current patterns.
3. Prefer standard library features.
4. Do not add dependencies unless clearly needed.
5. Avoid unnecessary abstractions.
6. Keep functions small.
7. Add tests for core logic.
8. Do not refactor unrelated files.
9. Do not commit secrets.
