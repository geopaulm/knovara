# DocuMind — Frontend Requirements

## 1. Overview

The DocuMind frontend is a simple web application that allows users to upload documents, view document processing status, and ask questions about uploaded content.

The frontend should be clean, responsive, and suitable for demonstrating the project in an Upwork portfolio.

## 2. Recommended Stack

* Next.js
* TypeScript
* Tailwind CSS
* React Hook Form
* Axios or Fetch API

## 3. Main Pages

### 3.1 Home Page

The home page should briefly explain what DocuMind does.

It should include:

* Product name
* Short description
* Primary button to open the chat page
* Secondary button to upload documents
* Simple feature highlights

Example feature highlights:

* Upload PDFs
* Ask questions
* Get answers with sources

### 3.2 Documents Page

The documents page should allow users to manage uploaded documents.

It should include:

* Upload document button
* Drag-and-drop upload area
* List of uploaded documents
* Processing status for each document
* Delete document option

Document statuses:

* Uploaded
* Processing
* Ready
* Failed

### 3.3 Chat Page

The chat page should allow users to ask questions about uploaded documents.

It should include:

* Question input box
* Submit button
* Loading state while waiting for a response
* Answer display area
* Source citations below each answer

Each source citation should show:

* Document name
* Page number, if available
* Relevant text excerpt
* Expand/collapse option for longer excerpts

If no documents are ready, the page should show a message asking the user to upload documents first.

## 4. User Interface Requirements

The UI should be:

* Simple
* Clean
* Responsive
* Easy to demo
* Suitable for screenshots

The layout should work well on:

* Desktop
* Tablet
* Mobile

The application should include basic navigation between:

* Home
* Documents
* Chat

## 5. API Integration Requirements

The frontend should call the backend APIs for:

* Uploading a document
* Listing documents
* Deleting a document
* Asking a question

Expected API routes:

`POST /api/documents`

`GET /api/documents`

`DELETE /api/documents/{id}`

`POST /api/chat`

The backend base URL should be stored in an environment variable.

Example:

`NEXT_PUBLIC_API_BASE_URL=http://localhost:8000`

## 6. State Management

For the MVP, complex state management is not required.

React local state is enough for:

* Upload progress
* Document list
* Chat messages
* Loading states
* Error messages

A global state library is not required for the MVP.

## 7. Error Handling

The frontend should display friendly messages for:

* Upload failure
* Unsupported file type
* Backend connection failure
* Document processing failure
* Empty question submission
* No answer found from documents

Example message:

“The uploaded documents do not contain enough information to answer this question.”

## 8. Loading States

The frontend should show clear loading states for:

* Uploading a document
* Loading document list
* Deleting a document
* Generating an answer

The chat submit button should be disabled while an answer is being generated.

## 9. Validation

The frontend should validate:

* File is selected before upload
* File type is PDF
* Question is not empty
* Question length is within a reasonable limit

Recommended question limit:

* Maximum 1,000 characters

## 10. Out of Scope for MVP

The following features are not required for the initial version:

* User login
* User registration
* Organisation/team management
* Real-time streaming responses
* Advanced chat history
* Dark mode
* Document preview
* PDF viewer
* Payment integration
* Admin roles

## 11. Portfolio Success Criteria

The frontend is complete when a user can:

1. Open the application
2. Upload a PDF
3. See the document status
4. Ask a question
5. View an AI-generated answer
6. See source citations used for the answer

The UI should provide enough polish to be used in screenshots for Upwork, GitHub, and portfolio pages.
